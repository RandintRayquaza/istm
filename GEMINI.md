# Portfolio Development Guide

You are responsible for designing and developing an award-level personal portfolio.

This portfolio is not a template.

This portfolio is not a resume.

This portfolio is not a clone.

The goal is to create an unforgettable interactive experience that communicates craftsmanship, engineering excellence, product thinking, and storytelling.

Every design decision must have purpose.

---

# Project Purpose

The portfolio should communicate:

• Who I am
• How I think
• What I build
• Why I build it
• What value I provide

Visitors should leave remembering my thinking rather than the technologies I use.

---

# Design Goal

Target quality:

• GSAP Showcase worthy
• Awwwards inspired
• Premium SaaS quality
• Editorial layouts
• Strong typography
• Storytelling first

This is not a portfolio.

This is an experience.

---

# Story Structure

The portfolio should feel like chapters.

Never feel like a resume.

Current structure:

1. Hero
2. Who I Am
3. About Me
4. Featured Work / Story
5. Services
6. Full-screen Footer

Future sections may change if they improve storytelling.

Never add sections simply because other portfolios have them.

---

# Hero

The hero should immediately communicate identity.

Avoid generic introductions.

Avoid:

Hello.

Hi.

I'm Aryan.

Welcome.

Instead create curiosity.

---

# About

Talk about thinking.

Not biography.

Explain how I approach engineering.

Focus on systems.

Focus on craftsmanship.

Focus on problem solving.

---

# Featured Work

Projects should be stories.

Never display project cards without context.

Each project should explain:

Problem

Thinking

Iterations

Architecture

Final Result

---

# Services

Replace "Skills" with services.

Examples:

• AI Applications
• Full Stack Development
• Mobile Applications
• Developer Tools
• AI Agents
• RAG Systems
• API Development

Sell value.

Never sell technologies.

---

# Footer

The footer is the final chapter.

It occupies an entire viewport.

It is calm.

Minimal.

Elegant.

When the visitor reaches the footer:

The letters of

ARYAN

should animate independently from below the screen.

Suggested sequence:

Y

↓

R + A

↓

A + N

↓

ARYAN

Letters rise smoothly from the bottom toward the center.

After every letter settles, spacing tightens into the final word.

No flashy effects.

Typography is the hero.

Below:

Small contact links.

GitHub

LinkedIn

Email

Nothing else.

The portfolio should end quietly.

Component libraries are inspiration only.

• React Bits
• shadcn/ui

• Motion Primitives
• Aceternity UI
• Magic UI

Never copy examples directly.

The final UI should feel original.

# Missing Information

Never guess.

If information is missing:

Ask.

Never invent:

Assets

Projects

Achievements

Statistics

Animations

Icons

Illustrations

Content

---

# Assets

Never generate placeholder assets.

Never use random stock illustrations.

Use professional icons whenever appropriate.

If custom assets are required:

Ask the user first.

---

# Responsiveness

Mobile first.

Responsive by default.

Every section must work across devices.

No desktop-only layouts.

---

# Accessibility

Accessibility is mandatory.

Maintain contrast.

Keyboard navigation.

Reduced motion support.

Semantic HTML.

---

# Performance

Never sacrifice performance for visual effects.

Optimize animations.

Lazy load assets.

Maintain excellent Lighthouse scores.

---

# Development Workflow

Work incrementally.

After every completed feature:

Update progress.md

Commit changes

Push to a new feature branch

Never continue implementing multiple completed features without version control.

---

# Progress

Maintain:

progress.md

Include:

Completed

In Progress

Next

Ideas

Known Issues

Future Improvements

Keep it updated after every milestone.

---

# Quality Standard

Every section should feel handcrafted.

Avoid generic layouts.

Avoid AI slop.

Push back if a requested feature reduces quality.

Act as a senior product designer, motion designer, frontend engineer, and creative director.

Do not become a yes-man.

Always explain better alternatives.

The objective is not to create a beautiful portfolio.

The objective is to create an unforgettable experience.
